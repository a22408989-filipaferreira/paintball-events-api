Making Of

Nesta página é apresentado todo o processo de desenvolvimento da API RESTful Paintball Events API, desde a modelação inicial até à implementação final utilizando Django Ninja.

Escolha do Tema

O tema escolhido foi a gestão de eventos de paintball. A ideia surgiu por ser uma área interessante e permitir criar várias entidades relacionadas entre si, adequadas para uma API RESTful.

DER

Foi elaborado um Diagrama Entidade-Relação em papel antes da implementação. Este permitiu identificar as entidades principais e as respetivas relações.

Models

Foram criadas três entidades principais: Event, Participant e Team. As relações foram implementadas através de Foreign Keys e ManyToMany.

Admin

O Django Admin foi configurado para permitir a gestão dos dados e inserção de registos de teste.

Schemas

Foram utilizados Schemas do Django Ninja para validar e serializar os dados trocados pela API.

API

Foram implementados endpoints CRUD completos para todas as entidades. Também foram adicionados filtros, ordenação e paginação.

Swagger

A documentação automática foi gerada através do Swagger, permitindo testar todos os endpoints diretamente no browser.

Dificuldades Encontradas

As principais dificuldades estiveram relacionadas com a configuração inicial do Django Ninja, as relações entre entidades e a implementação da paginação e filtros.
